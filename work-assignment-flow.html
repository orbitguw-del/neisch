Work Assignment — Flow

Storey · cascading task assignment, contractor → worker
LEVEL 1 · CONTRACTOR
Contractor
Creates a top-level task for a site (e.g. "Complete 2nd-floor slab") and assigns it to a Site Manager.
assigns task →
LEVEL 2 · SITE MANAGER
Site Manager
Opens the assigned task, breaks it into sub-tasks (shuttering, reinforcement, pour…) and assigns each to a Supervisor.
divides → sub-tasks
LEVEL 3 · SUPERVISOR
Supervisor
Opens each sub-task, divides the work among Workers, and logs a daily progress update — note + date-stamped photo — every day until the task is complete.
divides → worker tasks
LEVEL 4 · WORKERS
Workers
Each worker has a unit of work. Workers do not log in — the Supervisor records and updates their task status on their behalf.
⚠️ Workers are records, not app users. The Supervisor is the one who marks worker-level tasks done. (Worker logins could be a future option.)
Status lifecycle (every task at every level)
Pending ▶ In progress ▶ Submitted ▶ Done · Blocked
Pending — assigned, not started
In progress — work underway
Submitted — the assignee marked it finished — awaiting confirmation
Done — confirmed by the person who assigned it
Blocked — stuck (material shortage, weather, dependency)
⏱ Overdue — a derived flag: any task not Done past its due date. Not a manual status — the system sets it.
Confirmation / sign-off
• Marking a task finished does not close it — it moves to Submitted and waits for the person who assigned it.
• The Site Manager confirms a Supervisor's submitted task — after reviewing its photo timeline. Only then does it become Done.
• Same rule up the chain: the Contractor confirms a Site Manager's task; the Supervisor confirms a Worker's task.
• If the work isn't acceptable, the confirmer sends it back → returns to In progress for rework.
Same approve / reject pattern already used for site expenses — the assigner signs off on the assignee's work.
The confirm rule applies to all Supervisor work
Nothing a Supervisor records is final on its own — the Site Manager confirms it. Every supervisor entry is Submitted first, then Confirmed:
• Attendance — Supervisor marks the daily roster → Site Manager confirms → only then it counts toward wages & payroll.
• Daily logs — Supervisor files the log → Site Manager confirms.
• Tasks — Supervisor submits a finished task → Site Manager confirms.
• Site expenses — already works this way (pending → approved).
One consistent rule across the app: the Supervisor does the work, the Site Manager signs it off.
⚠️ Scope note: adding confirmation to attendance and daily logs changes existing modules — each needs a status column, a Site-Manager "confirm" screen, and payroll/reports must count only confirmed attendance. A bigger change than the task module alone — worth doing as its own phase.
Daily progress tracking
• Until a task is Done, the Supervisor logs a daily update on it: a short note on what was done + a date-stamped photo.
• Each task builds a day-by-day photo timeline — visible proof of how the work progressed.
• A task with no update for 2+ days is flagged Stalled — separate from "Overdue", it catches work that quietly stops.
• Managers and the Contractor scroll a task's timeline to see exactly what happened each day.
Stored as task updates (date · note · photo) — one row per day, like a per-task daily log.
Time-bound & delay tracking
• Every task is assigned with a due date by whoever assigns it (optional start date too).
• A task past its due date and not Done is flagged Overdue — "3 days late" in red.
• Delay rolls up: if any sub-task is overdue, its parent is marked At risk — so a delay deep in the tree is visible at the top.
• Dashboard shows a "Delayed tasks" count; Reports get an Overdue / On-time breakdown per site and per person.
Result: the Contractor sees at a glance which site, which manager, and which task is slipping — and by how many days.
Progress roll-up
As workers' tasks are marked done, the Supervisor's sub-task shows "4 / 6 done". As Supervisors finish, the Site Manager's task shows its sub-task progress. The Contractor sees the top task roll up the same way — one glance, top to bottom.
Who can assign to whom
• Contractor → assigns to Site Managers on the site
• Site Manager → assigns sub-tasks to Supervisors on the site
• Supervisor → assigns sub-tasks to Workers on the site
Each level can only assign one step down. Store Keeper is not in the work chain.
