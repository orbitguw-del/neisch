## Wall #1: Camera crashes the Android app

**What broke:** Tap "add photo" in a daily log on a real Android phone → app
crashes, no permission popup, no error message, back to home screen.

**What I assumed (first pass):** Missing CAMERA permission in AndroidManifest.
Fixed it in v1.1.2 with permissions + queries + file_paths.xml. Still crashed.

**What I assumed (second pass):** `CameraResultType.Uri` on Capacitor 8 Android
13+ returns a `capacitor://localhost/_capacitor_file_…` URI that the WebView
can't fetch. Switched to `Base64` in v1.1.3. Still crashed.

**What was actually true:** The native `@capacitor/camera@8.2.0` plugin is
incompatible with `targetSdkVersion 36` (Android 16). The crash happens inside
the plugin's native code, before any JS callback fires — confirmed by the
v1.1.4-debug build with explicit `alert()` markers at every step. Last alert
seen was `[3] about to call Camera.getPhoto`, never `[4]` returned.

**How I fixed it:** Stopped using `@capacitor/camera` on native. Routed
everything through the existing `webCapture()` path — a plain HTML file input
with `capture="environment"`. Same code that already worked on
storeyinfra.com mobile browser. v1.1.5 was 18 line additions, 38 line
deletions. Worked first try.

**What I learned:** When a dependency fights back across multiple fixes, the
correct move isn't another patch — it's bypassing the dependency. The
file-input flow loses zero functionality vs the native plugin (date-time
stamp still burns in, upload pipeline unchanged), and removes a dependency
that was actively breaking us. *Fewer moving parts = better* — exactly the
core philosophy in `CLAUDE.md`.

## Wall #2: New tester can't register — generic "non-2xx" error

**What broke:** Parth tried to create his company. Form showed "Edge Function
returned a non-2xx status code" with no further detail. Dead end.

**What I assumed (first pass):** Probably an RLS issue on the tenants insert,
or a missing env var. Was about to log into the edge function dashboard.

**What was actually true:** Parth's email already existed in `auth.users` from
a half-completed signup attempt back on 2026-05-07 (he was in the CSV as
"never signed in"). The function called `admin.auth.admin.createUser()`,
Supabase returned "user already exists", the function returned a 400, and
the supabase-js client wrapped that as a generic FunctionsHttpError without
exposing the real message.

**How I fixed it:** Two layers.
- `register-tenant` function now does a Step 0 lookup. If the email exists
  with a tenant attached: return 409 "please sign in." If it exists with no
  tenant (orphan): silently delete it and proceed with fresh registration.
- `authStore.signUp` now reads `error.context.json()` to extract the real
  server message instead of showing the generic supabase-js wrapper text.

**What I learned:** *Generic errors are silent budget killers.* Half the
time I waste on bugs is spent reverse-engineering what actually went wrong
because the error message hid it. Better error surfacing in 4 lines saves
hours later. Build error handling that tells the truth.

## Wall #3: Layout broken on phone — site title wraps one word per line

**What broke:** Site detail page on the installed app showed
"NH-37 / Extension / — / Furkating / to / Mariani" each word on its own
line. Description did the same. Third action button clipped off the right
edge of the screen.

**What I assumed:** A CSS class typo somewhere.

**What was actually true:** Classic flexbox squash. PageHeader had `flex
items-start justify-between` with no `min-w-0` on the title column and
`flex-shrink-0` on the action wrapper. On a narrow phone, the buttons
claimed their full width, the title column got squashed to its intrinsic
minimum, and the browser fell back to breaking on every space character.
Affected every page using PageHeader with an action — not just this one.

**How I fixed it:** PageHeader now stacks vertically on mobile
(`flex-col sm:flex-row`), the title column gets `min-w-0 flex-1`, and the
SiteDetail action row gets `flex-wrap` so buttons wrap to a second line
if 3 don't fit. One commit, every page using PageHeader is now correct.

**What I learned:** *Flexbox `min-width: auto` is a footgun on mobile.*
The default makes flex children refuse to shrink below their intrinsic
content width — which for long titles means letter-by-letter wrapping.
Always pair `flex-1` with `min-w-0` on text containers in responsive layouts.
A rule I now know, didn't know this morning.
